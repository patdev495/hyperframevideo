# Script Drafting Workflow

The News-to-Video Pipeline uses a portable, bring-your-own-AI approach for drafting video scripts. Instead of calling an LLM API automatically, you take the evidence from a local Production Run and use your preferred chatbot (e.g., ChatGPT, Claude, Grok, or Codex) to generate the script.

This guide explains how to generate a `SCRIPT.md` artifact from source evidence.

For the integrated DeepSeek-backed workflow, see [Pipeline Orchestrator Workflow](pipeline-orchestrator.md). The same boundary applies in both workflows: AI tools may only produce or repair **Source-Grounded Script** content, not storyboard, composition, voiceover, or render artifacts.

## 1. Locate Source Evidence

When you run the `hyperframe-video` CLI against a news URL, it creates a new Production Run directory under `.runs/<run-id>/`. Inside this directory, you will find a `source-evidence.json` file.

This JSON file contains the extracted article text, the title, the publication date, and any extraction warnings. This is the grounded factual data the AI must use to write the script.

## 2. Copy the Portable Prompt

The project maintains a carefully tuned prompt to instruct chatbots on how to write scripts for this pipeline.

Open [docs/prompts/script-drafting.md](../prompts/script-drafting.md) and copy the entire prompt text (everything inside the ````text ```` block).

## 3. Generate the Script

1. Open your preferred chatbot (ChatGPT, Grok, Claude, etc.).
2. Paste the prompt you copied in Step 2.
3. Open `.runs/<run-id>/source-evidence.json` and copy its entire contents.
4. Paste the JSON contents at the very bottom of the prompt (where it says `<PASTE SOURCE EVIDENCE HERE>`).
5. Send the message.

> [!IMPORTANT]
> The chatbot should output **`SCRIPT.md` only**.
> Do not ask the chatbot to produce a storyboard, animation plan, shot list, or HTML layout. `STORYBOARD.md` and HyperFrames composition work stay exclusively inside this repository and will be handled in later steps.

## 4. Replace the Draft

When you create a Production Run, the system generates a placeholder `SCRIPT.md` file with a `Status: draft` header. 

1. Copy the script generated by the chatbot.
2. Open the `.runs/<run-id>/SCRIPT.md` file in your editor.
3. Paste the chatbot's output to completely replace the placeholder content.

> [!NOTE]
> Ensure the output strictly follows the [authoritative script format](../artifacts/script.md).

## 5. Review and Approve

By default, the script header will be generated as:
```text
Status: draft
```

**The `Status` must remain `draft` until you have manually reviewed and approved the content.**

Read through the script to ensure:
- It is engaging and fits the target duration.
- It is accurately grounded in the source evidence.
- It does not invent facts or hallucinate claims.

Once you are satisfied, edit the file and change the status:
```text
Status: approved
```

Changing the status to `approved` acts as an approval gate. Future downstream pipeline commands (like voiceover generation or video rendering) will refuse to process the script unless it is explicitly marked as `approved`.
